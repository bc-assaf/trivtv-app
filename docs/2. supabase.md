## Local Supabase Development

See:
* [Local Development & CLI](https://supabase.com/docs/guides/local-development?queryGroups=package-manager&package-manager=pnpm)

```sh
pnpm add supabase --save-dev --allow-build=supabase
```

In your repo, initialize the Supabase project:
```sh
pnpx supabase init
```

Start the Supabase stack (make sure Docker is running):
```sh
pnpx supabase start
```

### Connecting the remote Supabase project to your local project

Login to Supabase:
```sh
npx supabase login
```
Link the remote project:
```sh
npx supabase link --project-ref your-project-ref-here
```

### Update `.env.local`

`.env.local`
```sh
DATABASE_URL="postgresql://postgres:postgres@127.0.0.1:54322/postgres"

PUBLIC_SUPABASE_URL="http://127.0.0.1:54321"
PUBLIC_SUPABASE_PUBLISHABLE_KEY="[local-key]"

```

#### Syncing Your Environments
Create a migration from remote:

```sh
npx supabase db pull
```

Apply it:
Refresh your local database and apply all migrations, including the one you just pulled.
```sh
npx supabase db reset
```
To overwrite local with remote:
1. Delete the files in your local supabase/migrations folder.
2. Run npx supabase db pull.
3. Run npx supabase db reset.


Push Local Changes to Remote

```sh
npx supabase db push
```

### Auth

Configure via the `config.toml` file.

Under `[auth]` section:
* Set `site_url` to `"http://127.0.0.1:5173"`

To apply changes, run:
```sh
npx supabase stop
npx supabase start
```

Get status:

```sh
npx supabase status
```

To create a type file describing the database schema for use with Supabase:

```sh
npx supabase gen types typescript --local > ./src/lib/db/database.types.ts

```
