
# Drizzle

## Configure Drizzle

In your drizzle.config.ts, point to your supabase/migrations folder if you want to keep all migrations in one place, or create a separate drizzle folder.

```ts
//drizzle.config.ts

import { defineConfig } from 'drizzle-kit';

export default defineConfig({
  schema: './src/lib/db/schema.ts',
  out: './supabase/migrations', // Storing migrations where Supabase expects them
  dialect: 'postgresql',
  dbCredentials: {
    url: process.env.DATABASE_URL!,
  },
});
```

## The Workflow
### Step 1: Create the migration
To generate a migration file based on the schema:
```sh
npx drizzle-kit generate [--name migration-name]
```

### Step 2: Apply locally

To re-create the DB based on migrations (this deletes auth users as well!):
```sh
npx supabase db reset
```

To apply new migrations to the local DB without losing the current users and data:
```sh
npx supabase db apply
```

Another option is using Drizzle `npx drizzle-kit migrate`)

### Push to Remote
When you are ready to move your local Drizzle changes to your production Supabase project:
```sh
npx supabase db push
```
